# Quiz Generator - Render Deployment Guide

## Overview
This document provides complete deployment instructions for the Quiz Generator application on Render.com.

## Prerequisites
- Render.com account
- GitHub repository with the project code
- Gemini API Key from [Google AI Studio](https://aistudio.google.com/)

## Render Environment Variables

### Backend Environment Variables
The following environment variables are required for the backend service:

| Variable | Source | Description |
|----------|--------|-------------|
| `SPRING_DATASOURCE_URL` | Auto-generated from Render PostgreSQL | PostgreSQL connection string |
| `SPRING_DATASOURCE_USERNAME` | Auto-generated from Render PostgreSQL | Database username |
| `SPRING_DATASOURCE_PASSWORD` | Auto-generated from Render PostgreSQL | Database password |
| `JWT_SECRET` | Auto-generated by Render | Secret key for JWT token signing |
| `GEMINI_API_KEY` | Manual (must be set) | Your Gemini API key from Google AI Studio |

### Frontend Environment Variables
The following environment variable is required for the frontend service:

| Variable | Value | Description |
|----------|-------|-------------|
| `VITE_API_BASE_URL` | `https://quiz-generator-backend.onrender.com/api` | Backend API URL |

**Note:** After deploying the backend, update the frontend `VITE_API_BASE_URL` with the actual backend URL provided by Render.

## Deployment Steps

### 1. Push Code to GitHub
```bash
git add .
git commit -m "Production deployment configuration"
git push origin main
```

### 2. Deploy on Render

#### Option A: Using render.yaml (Recommended)
1. Go to [Render Dashboard](https://dashboard.render.com/)
2. Click "New +" → "Blueprint"
3. Connect your GitHub repository
4. Select the repository containing `render.yaml`
5. Render will automatically create:
   - PostgreSQL database
   - Backend service (Spring Boot)
   - Frontend service (React/Vite)

#### Option B: Manual Deployment

##### Create PostgreSQL Database
1. Go to Render Dashboard → "New +" → "PostgreSQL"
2. Name: `quiz-generator-db`
3. Database: `quizgen`
4. User: `quizgen_user`
5. Region: Choose nearest region
6. Click "Create Database"

##### Create Backend Service
1. Go to Render Dashboard → "New +" → "Web Service"
2. Connect your GitHub repository
3. Build Command: `cd backend && mvn clean package -DskipTests`
4. Start Command: `cd backend && java -jar target/*.jar`
5. Environment Variables:
   - `SPRING_DATASOURCE_URL`: Copy from PostgreSQL database
   - `SPRING_DATASOURCE_USERNAME`: Copy from PostgreSQL database
   - `SPRING_DATASOURCE_PASSWORD`: Copy from PostgreSQL database
   - `JWT_SECRET`: Generate a secure random string (e.g., using OpenSSL)
   - `GEMINI_API_KEY`: Your Gemini API key
6. Click "Create Web Service"

##### Create Frontend Service
1. Go to Render Dashboard → "New +" → "Web Service"
2. Connect your GitHub repository
3. Build Command: `cd frontend && npm install && npm run build`
4. Publish Directory: `frontend/dist`
5. Environment Variables:
   - `VITE_API_BASE_URL`: Your backend service URL (e.g., `https://quiz-generator-backend.onrender.com/api`)
6. Click "Create Web Service"

### 3. Set Gemini API Key
After backend deployment:
1. Go to your backend service on Render
2. Navigate to "Environment" tab
3. Add `GEMINI_API_KEY` with your actual Gemini API key
4. Click "Save Changes"
5. The service will automatically restart

### 4. Update Frontend API URL
After backend deployment:
1. Copy the backend service URL from Render
2. Go to your frontend service on Render
3. Navigate to "Environment" tab
4. Update `VITE_API_BASE_URL` with the backend URL + `/api`
5. Click "Save Changes"
6. The service will automatically restart

## Local Development Setup

To run the application locally with PostgreSQL:

### Backend
```bash
cd backend
# Set environment variables or use application-local.properties
export SPRING_DATASOURCE_URL=jdbc:postgresql://localhost:5432/quizgen
export SPRING_DATASOURCE_USERNAME=quizgen_user
export SPRING_DATASOURCE_PASSWORD=your_password
export JWT_SECRET=your_jwt_secret
export GEMINI_API_KEY=your_gemini_api_key

mvn spring-boot:run
```

### Frontend
```bash
cd frontend
export VITE_API_BASE_URL=http://localhost:8080/api
npm run dev
```

## Troubleshooting

### Backend Fails to Start
- Check Render logs for errors
- Verify all environment variables are set
- Ensure PostgreSQL database is running
- Check database connection string format

### Frontend Cannot Connect to Backend
- Verify `VITE_API_BASE_URL` is correct
- Check CORS configuration in `SecurityConfig.java`
- Ensure backend is running and accessible

### Database Connection Issues
- Verify PostgreSQL credentials
- Check database is not in sleep mode (Render free tier)
- Ensure database URL format is correct

### Quiz Generation Fails
- Verify `GEMINI_API_KEY` is set correctly
- Check Gemini API quota
- Review backend logs for API errors

## Security Notes

- Never commit `.env` files or actual secrets to Git
- Use Render's environment variable management
- Rotate secrets periodically
- Enable HTTPS (Render provides this automatically)
- Review and update CORS settings for production

## Cost Considerations

For production use, consider upgrading to paid plans to avoid:
- Database sleep delays
- Service spin-up time
- Bandwidth limitations
